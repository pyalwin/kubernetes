You might expect the smallest thing Kubernetes runs to be a container. It isn't —
it's a **Pod**. A Pod is the smallest deployable unit, a wrapper around *one or
more* containers that are always scheduled together on the same node and share
some of their environment. Most Pods hold a single container, but the wrapper
exists because sometimes you need a few containers to act as one tight unit.

## What the containers in a Pod share

Containers in the same Pod share two important things:

- **A network namespace.** The whole Pod gets *one* IP address. Containers
  inside reach each other over `localhost` and must agree on ports, just like
  processes on a single host. From outside, you talk to the Pod's IP, not the
  individual containers.
- **Volumes.** Containers in a Pod can mount the same volume, so one can write a
  file and another can read it.

This shared context is exactly what makes the **sidecar** pattern work. A
sidecar is a helper container next to your main app in the same Pod — a log
shipper tailing a shared log directory, a proxy handling TLS, or an agent
syncing config. Because they share network and storage, the sidecar augments the
main container without the two needing to know each other's IP.

## Pod lifecycle phases

A Pod moves through a small set of high-level **phases**:

- **Pending** — accepted by the cluster but not yet running (image pulling, or
  waiting for the scheduler to place it).
- **Running** — bound to a node with at least one container started.
- **Succeeded** — all containers exited 0 and won't restart (typical for Jobs).
- **Failed** — all containers terminated and at least one failed.

The `restartPolicy` (`Always`, `OnFailure`, `Never`) controls what happens when
a container inside exits.

## Why you rarely create bare Pods

A bare Pod has nobody watching it. If it crashes in a way it can't recover from,
or its node dies, *nothing recreates it* — it's gone. That's why in practice you
almost always create Pods through a **controller** (a Deployment, StatefulSet,
Job, or DaemonSet). The controller treats Pods as cattle: it watches the desired
count and replaces any that disappear, giving you self-healing for free.
